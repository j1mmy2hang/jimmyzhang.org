## The vision

For a long time, malleable software was framed as the more empowering vision because it gave users more control.

The idea that users should be able to shape their own tools, define their own workflows, and gradually turn software into something uniquely theirs. It is an appealing vision. It treats software as a medium and the user as a creator.

## The Problem -- Tool vs Outcome

Most people do not want or have time, or capacity to build their own tools. They want something they can rely on, share with others, and return to later without having to reconstruct how it was put together. Tools are for doing the work, not for constantly tuning the tool itself. The tool building is a side quest, not the main driver of any real work.

The idea of malleable, or customizable software kind of loses its purpose. You don’t need a complicated middleware system of malleable blocks or ways to customize. 

## Solution

Agents 可以解释用户意图，自动适应, 实现真正 personalized software

Users should get better outcomes without having to become architects of their own tools.

> The real promise of this era is not that everyone shapes and customizes their own tools with malleable systems.
> 
> It is that agents become capable enough that we no longer need to.
