> [!Principle]
> 是否能有效，快速，便宜，稳定地解决 agent 问题。

## 有效

得解决问题

传统营销的核心是抢占注意力——广告、落地页、冷邮件，全是为会分心、会冲动消费的人类设计的。

智能体不浏览，只查询。它的决策逻辑极其简单：**你能解决我的问题吗？多快？多少钱？稳不稳？**

## 便宜

每遇到一个子任务，智能体都会做一个判断：自己算，还是花钱买现成答案？
成本必须低于 agent 自己计算的成本
专业服务永远做到通用智能体便宜。

但要注意：随着模型越来越便宜，部分服务会被智能体重新内化。能存活下来的，是智能体无法自己复制的东西：**专有数据集、实时数据流、依赖硬件的计算**（如图像生成）。

卖的是智能体自己算不出来的东西。

## 稳定

**信任变成了可量化的指标。** 
品牌不会消失，它变成了可靠性评分——在线率、响应延迟、准确率、输出来源。能证明自己输出准确的服务，会击败更便宜但不透明的竞争者。

可证明的可靠性 — 公开在线率、延迟分位数、准确率；最好在响应中附带置信度分数

## Agent friendly

- 服务发现必须是机器可读的 -- JSON格式的服务描述
- 价格写在协议里，按请求计费
- 可自动化的注册流程 -- 让智能体无需人工就能完成从发现到付款的全流程
